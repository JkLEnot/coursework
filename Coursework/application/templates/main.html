Регулярные обновления безопасности

Каждые шесть месяцев Ubuntu Server предоставляет обновления обслуживания, как и его настольный аналог. Он также предлагает стандартное окно обеспечения безопасности (SSM) на пять лет.

Масштабирование

С сервером вы можете установить минимальную или стандартную установку. Более того, вы можете развернуть его на различных машинах, и он предлагает широкие возможности горизонтального масштабирования. Это означает, что вы адаптируете его к вашим потребностям по мере развития.

Использование ресурсов

Ubuntu в целом завоевал репутацию благодаря эффективному использованию ресурсов. К счастью, это качество распространяется и на Убунту сервер. Действительно, поскольку он отказывается от GUI, его аппаратные требования относительно скромны.

Шаг 1. Загрузка образа.

В данном примере установка OS Ubuntu Server будет происходить через запись образа на флешку. Сначала нужно скачать установочный образ (ISO) и записать его на флешку. Скачать Ubuntu Server ISO вы можете на официальном сайте. На данный момент доступны версии 20.04.1 LTS и 20.10. Процесс установки для них мало отличается. Но первая - это стабильная версия и именно ее рекомендуется использовать для серверов.

Шаг 2. Запись на флешку.

Для записи образа на флешку можно использовать утилиту UltraISO, для начала нажмите кнопку 'Файл' -> 'Открыть...' после чего откройте скаченный ISO файл. Далее нужно нажать на кнопку 'Самозагрузка' -> 'Записать образ Жесткого диска...', после нужно выбрать флешку на которую будет производиться запись, метод записи оставьте поу молчанию, и теперь нажмите 'Записать' -> 'Да', после чего начнется запись образа на флешку.

Шаг 3. Настройка биоса.

Я не буду описывать как настроить BIOS на загрузку с флешки, т.к все зависит от версии BIOS и производителя материнской платы. Просто перезагрузите компьютер, откройте меню BIOS и выставьте вашу флешку первой среди устройств загрузки.

Шаг 4. Начало установки.

После перезагрузки пк появится окно загрузки. Настоятельно рекомендуется иметь проводное подключение к Интернет на сервере, на котором вы будете устанавливать «Ubuntu». Это позволит оперативно обновить программное обеспечение непосредственно в процессе инсталляции.

Шаг 5. Выбор языка.

После не долгого запуска всех сервисов установочного образа должно появиться окно с меню выбора языка.

Можно выбрать любой из списка, но для установки серверной операционной системы лучше выбрать английский, так как в случае возникновения ошибки намного проще и быстрее будет найти её описание в англоязычном сегменте Интернета, но я все же выбрал русский язык. Выбрав требуемый язык в меню продолжите установку.

Шаг 6. Выбор языка раскладки клавиатуры.

В ходе дальнейшей установки будет предложено выбрать раскладку клавиатуры.

Поскольку мы устанавливаем серверную операционную систему и работать в основном мы будем в консольном режиме, выберем английский язык по умолчанию. Подтверждаем, выбрав пункт меню «Готово», и продолжаем.

Шаг 7. Настройка сети.

Далее будет предложено настроить сетевые подключения. В данном конкретном случае интерфейс автоматически подхватил IP-адрес с DHCP-сервера (10.0.2.15/24)

Выбрав требуемый сетевой интерфейс, можно настроить его вручную, задать статический IP-адрес для IPv4 или IPv6, создать VLAN или просто посмотреть детальную информацию о текущем состоянии подключения.

Шаг 8. Настройка прокси.

В ходе дальнейшей установки будет предложено указать прокси-сервер на тот случай, если сервер, где производится разворачивание «Ubuntu», не имеет прямого подключения к Интернету либо существуют ограничения на доступ к репозиториям «Canonical», устранить которые можно только с помощью внешнего прокси.

В данном случае этот режим нам не нужен. Подтверждаем, выбрав пункт меню «Готово» и оставив поле «Адрес прокси-сервера» пустым.

Шаг 9. Выбор зеркала.

Серверы репозиториев — зеркала с установочными пакетами Ubuntu, расположенные по всему миру (список можно посмотреть на официальном сайте). Установщик сам определяет наиболее близко расположенное к нему зеркало для обеспечения лучшей скорости и стабильности скачивания необходимых компонентов. Однако иногда стоит указать URL-адрес зеркала вручную, заполнив поле «Mirror address».

В нашем случае оставляем без изменений. Продолжаем, выбрав пункт меню «Готово».

Шаг 10. Настройка дисков.

Самый простой способ разметки диска, подходящий новичкам — предоставление установщику всего диска в распоряжение по умолчанию (Use an entire disk), но даже если вы выберите его, во время разметки можно сбросить настройки.

Однако лучше всего это всё сделать вручную, так мы будем контролировать процесс и сможем более рационально использовать дисковое пространство.

Отмечаем клавишей «Пробел» флажок «Custom storage layout» и подтверждаем, выбрав пункт меню «Готово».

Выбираем из списка «AVAILABLE DEVICES» диск, который будем использовать.

В контекстном меню выбираем пункт Add GPT Partition.

Для начала создадим SWAP-раздел, выполняющий ту же роль, что и файл подкачки в «Windows». Обычно задают размер, вдвое больший объем оперативной памяти компьютера. Размер указывается в гигабайтах. В нашем случае ОЗУ — 4 Гб, соответственно, «SWAP» будет 8 Гб.

Для начала создадим SWAP-раздел, выполняющий ту же роль, что и файл подкачки в «Windows». Обычно задают размер, вдвое больший объем оперативной памяти компьютера. Размер указывается в гигабайтах. В нашем случае ОЗУ — 4 Гб, соответственно, «SWAP» будет 8 Гб.

Указываем выбранный размер в поле «Size». В раскрывающемся списке «Формат» выбираем тип «swap » и далее выбираем пункт меню «Создать».

Продолжая разметку, повторим действия с выбором диска. По умолчанию будет предложена точка монтирования, распространяемая на весь корневой раздел /.

Как видим, у нас готовы для создания SWAP-раздел и том с используемой для него файловой системой Ext4 и точкой монтирования /. Выбираем пункт меню «Готово».

«Внимание! Производимые изменения на диске, на который устанавливается Ubuntu 20.04 LTS Server (Focal Fossa), станут необратимыми, вся информация на нём будет затерта.»

Продолжаем установку, выбираем пункт меню «Продолжить».

Шаг 11. Создание пользователя и настройка имени сервера.

В следующем окне установщика необходимо указать имя пользователя в поле «Ваше имя», имя сервера в поле «Your server’s name», логин для входа в систему в поле «Введите имя пользователя» и дважды ввести пароль учётной записи в поля «Задайте пароль» и «Повторите пароль».

Шаг 12. Установка OPENSSH-Сервера.

Для удалённого управления сервером Linux используется специальный защищённый SSH-протокол (Secure Shell).

Если вы не планируете все операции по настройке сервера делать локально, настоятельно рекомендуем установить OpenSSH. Отмечаем клавишей «Пробел» флажок «Install OpenSSH Server».

Опционально в этом же окне для авторизации по протоколу SSH можно добавить SSL-ключи. Однако пока оставим всё по умолчанию, ключи можно создать или импортировать после установки.

Шаг 13. Выбор и установка приложений.

В следующем окне можно сразу же установить популярные серверные окружения и приложения: «Docker», «Kubernetes», различного рода «бенчмарки» или инструменты для взаимодействия с облачными провайдерами («aws-cli», «google-cloud-sdk» и др.).

Пока не будем устанавливать,. при необходимости это можно сделать после. Для продолжения выбираем пункт меню «Готово».

Шаг 14. Установка системы.

В следующем окне мы будем наблюдать за процессом установки ОС и выбранных нами пакетов программ.

Шаг 15. Завершение установки.

После того, как ОС и все пакеты будут проинсталлированный, появится окно с сообщением «Installation Complete!».

Выбираем пункт меню «Перезагрузить сейчас» для выполнения перезагрузки компьютера, параллельно извлекаем носитель с дистрибутивом «Ubuntu 20.04 LTS Server».

Шаг 16. Вход в систему.

После перезагрузки системы нас попросит авторизоваться с созданной прежде учетной записи.

После авторизации откроется стандартная консоль «Linux-сервера».

Шаг 17. Первая команда.

После увлекательного процесса установки «Ubuntu Server» вы можете ввести свою первую команду lsb_release -a, например проверить версию «Ubuntu Server».
